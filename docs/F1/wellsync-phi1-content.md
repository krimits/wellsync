## ΠΕΔΙΟ 4 — Πρόβλημα που Αντιμετωπίζεται

```
Οι φοιτητές και νέοι ενήλικες (18–30) προσπαθούν να διατηρήσουν
υγιεινές συνήθειες με ακανόνιστο πρόγραμμα, εξεταστικές περιόδους
και αυξημένο stress. Οι υπάρχουσες εφαρμογές ευεξίας λειτουργούν
σε silos: το MyFitnessPal καλύπτει μόνο διατροφή, το Strava μόνο
προπόνηση, το Headspace μόνο mental health. Καμία εφαρμογή δεν
συσχετίζει και τους τρεις τομείς ούτε προσαρμόζει τις συστάσεις
βάσει των ατομικών δεδομένων κάθε χρήστη.

Αποτέλεσμα: ο χρήστης γνωρίζει τι "πρέπει" να κάνει αλλά δεν
ξέρει γιατί χθες η προπόνησή του ήταν καλή και σήμερα όχι.
```

## ΠΕΔΙΟ 5 — Προτεινόμενη Λύση

```
Το WellSync αντιμετωπίζει τον χρήστη ως ένα σύστημα: συλλέγει
καθημερινά δεδομένα από τρεις τομείς (σωματική δραστηριότητα,
διατροφή, ψυχική κατάσταση) και εφαρμόζει Machine Learning για
να ανακαλύψει τα ατομικά patterns κάθε χρήστη.

Κεντρική λειτουργία: κάθε πρωί το σύστημα υπολογίζει το "Readiness
Score" (0–100) βάσει του check-in, και ένα AI agent — που εκτελείται
αυτόματα στο παρασκήνιο — παράγει εξατομικευμένη σύσταση σε φυσική
γλώσσα (π.χ. "Κοιμήθηκες 5.5h και χθες ήταν high-intensity. Σήμερα
προτείνω yoga 20' και γεύμα με πρωτεΐνη — αποφύγετε το HIIT μέχρι
να επιστρέψει ο ύπνος σου πάνω από 7h.").

Αυτή η σύσταση δεν είναι pre-written template: παράγεται από LLM
(AI γλωσσικό μοντέλο) που έχει πρόσβαση στα τελευταία 7 days
δεδομένων του χρήστη.
```

## ΠΕΔΙΟ 6 — Καινοτομία / Πρωτοτυπία

```
Το WellSync διαφέρει από τις υπάρχουσες λύσεις σε τρία επίπεδα:

1. Cross-domain ML Correlation
   Αντί να αναλύει κάθε τομέα (ύπνος, προπόνηση, διατροφή) ξεχωριστά,
   το WellSync εφαρμόζει per-user Ridge Regression για να ανακαλύψει
   ατομικά patterns που συνδέουν τους τομείς (π.χ. συσχέτιση μεταξύ
   ύπνου και απόδοσης προπόνησης για τον συγκεκριμένο χρήστη).

2. Agentic Event Loop με AI
   Ένα σύστημα event-driven agents εκτελείται αυτόματα κάθε πρωί:
   - Scheduler πυροδοτεί event → τοποθετείται σε queue → gateway
     δρομολογεί στον κατάλληλο agent
   - Ο MorningRecommendationAgent συνδυάζει τα ML αποτελέσματα με
     κλήση σε AI (Large Language Model) για να παράγει natural language
     σύσταση προσαρμοσμένη στον χρήστη
   - Ο αρχιτεκτονικός σχεδιασμός υποστηρίζει εύκολη επέκταση με
     νέους agents (εβδομαδιαίος απολογισμός, ειδοποιήσεις, κ.λπ.)

3. Progressively Personalized
   Νέοι χρήστες λαμβάνουν rule-based συστάσεις άμεσα. Μετά τις
   14 μέρες δεδομένων ενεργοποιείται το personalized ML μοντέλο.
   Ο χρήστης βλέπει την πρόοδο: "Personalized mode active – 18 days".

4. Evidence-Based RAG (Retrieval-Augmented Generation)
   Πριν από κάθε κλήση στο LLM, το σύστημα ανακτά τα πιο σχετικά
   επιστημονικά αποσπάσματα από curated knowledge base (ACSM, WHO,
   Sleep Foundation, ISSN) μέσω semantic search (pgvector cosine
   similarity). Η σύσταση δεν βασίζεται μόνο στα δεδομένα του
   χρήστη — ενισχύεται από τεκμηριωμένες επιστημονικές οδηγίες.
```

Οι φοιτητές AUEB αποτελούν την ιδανική αγορά για MVP — accessible,
tech-savvy, και με clear pain point (εξεταστικό stress + irregular sleep).

## ΠΕΔΙΟ 9 — Τεχνολογία / Αρχιτεκτονική

```
Πλατφόρμα: Web application

Backend:
  - Python 3.10 + FastAPI (REST API)
  - PostgreSQL + pgvector extension (relational + vector database)
  - scikit-learn — Ridge Regression per user (ML personalization)
  - sentence-transformers/all-MiniLM-L6-v2 — semantic embeddings (RAG)
  - APScheduler — scheduled event firing
  - asyncio Queue + EventGateway — event routing architecture
  - Anthropic API (Claude AI) — natural language recommendations

Frontend:
  - React 18 + Vite (modern SPA)
  - Chart.js — trend visualization

Deploy:
  - Docker Compose (PostgreSQL + Backend + Frontend σε 3 containers)
  - Εκτελείται τοπικά με: docker compose up --build

Αρχιτεκτονική Agentic Loop:
  Scheduler → Queue → Gateway → Agent → Claude API → DB → Frontend
```

## ΠΕΔΙΟ 12 — Σύνοψη Καινοτομίας (1 παράγραφος για executive summary)

```
Το WellSync είναι μια AI-powered web εφαρμογή ευεξίας για φοιτητές
που λύνει ένα συγκεκριμένο πρόβλημα: οι υπάρχουσες εφαρμογές δεν
συνδέουν ύπνο, προπόνηση και διατροφή — ούτε προσαρμόζονται στον
συγκεκριμένο χρήστη. Το WellSync το κάνει μέσω ενός event-driven
agentic loop: κάθε πρωί ένας AI agent διαβάζει τα δεδομένα του
χρήστη, εκτελεί ML ανάλυση, ανακτά σχετικές επιστημονικές οδηγίες
(ACSM, WHO, Sleep Foundation) μέσω RAG (pgvector semantic search) και
κάνει κλήση σε LLM (Claude AI) για να παράγει εξατομικευμένη,
τεκμηριωμένη σύσταση σε φυσική γλώσσα. Η αρχιτεκτονική
(Scheduler → Queue → Gateway → Agents) υποστηρίζει επεκτασιμότητα,
ενώ ο συνδυασμός ML + RAG + LLM αποτελεί σαφή τεχνολογική
διαφοροποίηση από τον ανταγωνισμό.
```
