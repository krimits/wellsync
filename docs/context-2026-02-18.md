## 3. Η Ιδέα (approved)

### Πρόβλημα
Οι υπάρχουσες wellness εφαρμογές είναι siloed:
- MyFitnessPal → μόνο nutrition
- Strava → μόνο workout
- Headspace → μόνο mental health

Κανείς δεν συσχετίζει και τους τρεις τομείς ή προσαρμόζει τις συστάσεις βάσει ατομικών patterns.

### Λύση
WellSync: web app για φοιτητές/νέους (18–30) που:
1. Συλλέγει **daily check-in** (ύπνος, διάθεση, ενέργεια, στρες)
2. Καταγράφει **workouts** και **γεύματα**
3. Υπολογίζει **Readiness Score** (0–100)
4. Εφαρμόζει **per-user ML model** για να ανακαλύπτει ατομικά patterns
5. Δίνει **προσαρμοσμένες συστάσεις** ημέρας

### Κεντρική Καινοτομία
Cross-domain ML correlation: "όταν κοιμάσαι <6h + στρες εξετάσεων, η απόδοσή σου πέφτει 40% — σήμερα ξεκουράσου".

## 4. Αρχιτεκτονικές Αποφάσεις

### Tech Stack
| Layer | Τεχνολογία | Αιτιολογία |
|-------|-----------|-----------|
| Backend API | FastAPI (Python 3.10+) | Ταιριάζει με ML skills, async, γρήγορο |
| ML Engine | scikit-learn (Ridge Regression) | Ελαφρύ, interpretable, no GPU needed |
| Database | PostgreSQL | Relational data, mature, Docker-friendly |
| Frontend | React 18 + Vite | Modern, component-based |
| Charts | Chart.js + react-chartjs-2 | Trend graphs για insights |
| Auth | JWT (python-jose + passlib) | Stateless, industry standard |
| Deploy | Docker Compose | Εύκολο setup για Φ4 demo |

### ML Στρατηγική
- **Cold start** (<14 days data): rule-based weighted readiness formula
- **Personalized** (≥14 days): Ridge Regression per user (predicts RPE from check-in features)
- **Insights**: Pearson correlation (sleep ↔ performance, mood ↔ performance)
- **Storage**: pickle files per user στο `ml_models/` directory

### Βασικά Endpoints
```
POST /auth/register
POST /auth/login
POST /checkin                    → υπολογίζει readiness score
GET  /checkin/history
POST /logs/workout
POST /logs/meal
GET  /logs/workouts
GET  /logs/meals
GET  /recommendation/today       → κύρια λειτουργία
GET  /insights/weekly            → correlations + trend data
```

### Database Schema (key tables)
```
users     → id, email, name, password_hash, created_at
checkins  → id, user_id, date, sleep_hours, sleep_quality, mood, energy, stress, readiness_score
workouts  → id, user_id, date, type, duration_min, rpe
meals     → id, user_id, date, meal_type, quality, notes
```

## 5. User Flow (Core Loop)

```
Πρωί
  └─ Daily check-in (2 min)
       sleep_hours [0–12] · sleep_quality [1–5] · mood [1–5] · energy [1–5] · stress [1–5]
       ↓
       ML → Readiness Score (0–100)
       ↓
       Home screen: recommendation card

Μέσα στη μέρα
  ├─ Log Workout: type · duration_min · RPE [1–10]
  └─ Log Meal: meal_type · quality [1–5] · notes

Βράδυ
  └─ Insights: trend graphs · correlation cards
```
